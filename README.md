# Speculative Decoding from Scratch (DSpark)

Companion code for the video: build lossless **speculative decoding** from an empty
file, then build and train DeepSeek's real **DSpark** drafter — and benchmark it
honestly against a dumb baseline. Torch only, runs on a laptop **CPU**, no GPU, no
downloads beyond one small pretrained model (`HuggingFaceTB/SmolLM2-135M`).

**📖 Prefer text? [LESSON.md](LESSON.md) is the full course as a written,
follow-top-to-bottom tutorial** — diagrams, tasks, code, and the measured
outputs at every step.

## Run it

```bash
pip install -r requirements.txt
python check_lossless.py     # proves speculative output is byte-identical to plain decoding
python train_drafter.py      # distills the DSpark drafter (~1 min CPU), writes drafter.pt
python eval_accept.py         # the honest acceptance benchmark
```

## What's here
| file | chapter | what it is |
| --- | --- | --- |
| `spec_decode.py` | 1–2 | baseline greedy decode + a dumb bigram drafter + the **lossless verifier** |
| `check_lossless.py` | 2 | asserts byte-identical output on several prompts |
| `dspark_drafter.py` | 3 | DSpark's drafter from scratch: parallel backbone + low-rank Markov head, reusing the frozen target's tied embedding/head |
| `data.py` | 4 | tiny distillation corpus |
| `train_drafter.py` | 4 | distill the drafter from the target's own greedy tokens |
| `eval_accept.py` | 5 | acceptance benchmark: bigram vs raw vs trained, all asserted lossless |
| `drafter.pt` | — | trained drafter weights (regenerated by `train_drafter.py`) |

## The honest result (measured here · SmolLM2-135M · CPU · greedy · all lossless)
| prompt set | bigram (free) | dspark trained | dspark raw |
| --- | --- | --- | --- |
| repetitive (code/technical) | **47%** | 32% | 0% |
| novel prose (held-out) | **22%** | 5% | 0% |

The zero-training n-gram ("prompt-lookup decoding") *beats* our from-scratch neural
drafter on this small greedy setup — a model's greedy output repeats itself, and the
n-gram looks those repeats up for free. Training still did a real thing (0% → 32%,
loss ~12.9 → 0.14) and is always lossless. The real DSpark wins where it's used: a much
stronger backbone and a **large** target where the drafter's cost is negligible and
repetition is rarer. Benchmark against the dumb baseline — always.
